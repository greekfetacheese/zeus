## 2026-06-30 - Witness Mapping Complete

`build_unshield_proof_request` now exists and maps real engine data (nullifiers, merkle proofs from scanner, input randoms/values, change notes, keys) into the exact `ProofRequest` structure expected by the JS sidecar and the Railgun Groth16 circuit.

Usage example (once you have notes in the scanner):

```rust
let prepared = engine.prepare_unshield(recipient, token, amount, use_broadcaster)?;
let proof_request = engine.build_unshield_proof_request(&prepared, Some("01x01"))?;

// Then
let client = RailgunProverClient::start(sidecar_path).await?;
let proof = client.prove_with_inputs(proof_request).await?;
```

The mapping is the bridge between the Rust Railgun logic and the ZK sidecar.
